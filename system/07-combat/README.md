# 戦闘ルール

- このドキュメントでは、セッション中に生じる戦闘についてのルールを解説します

---

## 戦闘の流れ

戦闘は下記の流れで進行する。

1. 遭遇とイニシアチブ
    - PCとエネミーが遭遇し、戦闘が開始される
2. エリアの配置とラウンドの進行
    - PCとエネミーを戦場に配置し、戦闘を進行する
3. 戦闘終了
    - 戦闘が終了し、戦利品などを獲得する

---

### 1. 遭遇とイニシアチブ

- 戦闘は、エネミーと遭遇した際に発生する
- 遭遇した際の状況や戦場、エネミーにより、有利な遭遇か、不利な遭遇かが決まる
- イニシアチブロールを行い、PCとエネミーの先手・後手を決定し、戦闘を開始する

#### イニシアチブロール

- 戦闘開始時、戦闘に参加するすべてのPCが、敵のイニシアチブ値を目標値に、イニシアチブ判定(LUC)を行う
    - イニシアチブ判定はLUCでのロールとなる
    - 一部の種族は、特性によって他のステータスを使用して判定することができる
    - 判定については、 [../06-adventuring/01-exploration.md](../06-adventuring/01-exploration.md) も参照すること
- イニシアチブ判定に誰か一人でも成功した場合は、PC側が先攻となる
    - 全員が失敗した場合はエネミーが先攻となる

> LUC判定:
> - 遭遇とはすべてが偶発的であり、その巡り合いの形は運によって定まる
> - それがセッション上の計画的遭遇であったとしても、PCにとっては偶発的である
>
> イニシアチブロールへの修正:
> - PCが地形的に不利なシチュエーションなどでは、-1dや-2dのペナルティを与えても良い
> - 奇襲ほどの有利状況ではないが、ある程度の不意を突いた遭遇であれば、状況に応じて+1dや+2dのボーナスを与えても良い
>
> 修正の例:
> - 扉の奥に陣取っている魔族の群れ。PCたちの侵入に気がついているため、PC側は-2dのペナルティを受けての遭遇となる
> - 周囲を警戒している魔族。物陰に潜むPCたちには気がついていないが、警戒はしているため、奇襲できない。PC側は+2dのボーナスを得ての遭遇となる

#### エネミーのデータ

- 遭遇時、GMはエネミーの情報をPLに開示する
- PLはエネミーのHPやMP、ステータスや行動表を把握しながら戦闘を行って良い
- エネミーのデータは [../08-enemies/](../08-enemies/) を参照すること

> エネミーへの知識判定の有無について:
> - ルール上は、エネミーの正体の看破に際し、知識判定は不要である
> 
> エネミー情報の非開示:
> - GMの判断で、エネミー情報を非公開としても良い
> - この場合、エネミーの匿名性を考慮し、PCに目標値3～5程度で知識判定を実行させると良い
>
> エネミーの創作:
> - GMは、ルールブック上にないエネミーを創作し、セッションに登場させても良い

---

### 2. エリアの配置とラウンドの進行

- エリアのルールに従い、GMとPLはキャラクターを戦闘エリアに配置する
- ラウンドルールに従い、キャラクターを操作し、戦闘を進行する

#### 2-1. 戦場のエリア構成

- 戦場は4つのエリアで構成される
- 各陣営は前衛と後衛の2つのエリアを持つ

```
[ 味方後衛 ] — [ 味方前衛 ] — [ 敵前衛 ] — [ 敵後衛 ]
```

> エリアの大きさとエリア間の距離:
> - 1つのエリアは10m程度の幅で構成される
> - エリアとエリアの間は密接している

---

#### 2-2. ラウンドの進行

- 戦闘は「ラウンド」の単位で進行していく
- ゲーム内での1ラウンドの経過時間は10秒である
- 「ラウンド」は全キャラクターの行動が終了するまでの1サイクルを指す
- 1ラウンドは下記の4つの「ターン」で構成される
- すべてのターンが終了するとラウンドが終了となり、次のラウンドを開始する

| 順番 | ターン         | 内容                                                  |
| ---: | :------------- | :---------------------------------------------------- |
|    1 | 先攻前衛ターン | 先攻陣営の前衛エリアにいるキャラクターが各1回行動する |
|    2 | 後攻前衛ターン | 後攻陣営の前衛エリアにいるキャラクターが各1回行動する |
|    3 | 先攻後衛ターン | 先攻陣営の後衛エリアにいるキャラクターが各1回行動する |
|    4 | 後攻後衛ターン | 後攻陣営の後衛エリアにいるキャラクターが各1回行動する |

ラウンドの進行順序:

$$
1ラウンド = 先攻前衛ターン → 後攻前衛ターン → 先攻後衛ターン → 後攻後衛ターン
$$

ターンのルール:
- 各キャラクターが行動できるターンは、ラウンド開始時に決定する
- エリアを移動した場合、そのラウンド中のそのキャラクターのターンは、移動前のエリアに準じる
- 各ターン内でのキャラクターの行動順は自由
    - PCのターンでの行動順は、PLが互いに相談して決めて良い

##### 移動したラウンド内での行動ターンについて

- 各キャラクターのターンは、常にラウンドの開始時に決定する
- 前衛にいるキャラクターが、自陣前衛のターンで自陣後衛に移動しても、そのラウンドの自陣後衛ターンでは行動できない
    - 次のラウンドからは、自陣の後衛ターンで行動する
- 自陣が先攻で、自陣後衛にいるキャラクターが、後攻前衛ターンで自陣前衛に強制移動させられた場合、先攻後衛ターンで自陣前衛のエリア上で行動する
    - 次のラウンドからは、自陣の前衛ターンで行動する

##### 敵陣営のエリアに移動した場合の行動ターンについて:

- 自陣の前衛・後衛ターンに準じて行動する
- 自陣が先攻で敵の後衛に移動していた場合、先攻後衛ターンで行動する (敵のターンでは行動しない)

##### 魔法やアイテムの効果時間について

- 「効果はXラウンド持続する」と書かれた効果は、ラウンド終了時に1ラウンド経過したものとする

例:
- 先攻前衛ターンで1ラウンド持続の効果を使用した場合、そのラウンドの終了時に効果が切れる (4ターン持続)
- 後攻後衛ターンで1ラウンド持続の効果を使用した場合も、そのラウンドの終了時に効果が切れる (1ターン持続)
- 先攻前衛ターンで3ラウンド持続の効果を使用した場合、3ラウンド目の終了時に効果が切れる (12ターン持続)
- 後攻前衛ターンで3ラウンド持続の効果を使用した場合、3ラウンド目の終了時に効果が切れる (9ターン持続)
- これらの持続効果は、ラウンド後半で使用するほど、実質的な効果時間が短くなる

---

#### 2-3. 行動と予備行動

- 各キャラクターは、自身のターンの間に「行動」と「予備行動」をそれぞれ1回ずつ実行できる

| 用語     | 説明                                                                           |
| :------- | :----------------------------------------------------------------------------- |
| 行動     | 各キャラクターがターン中に実行できる1回の動作 (攻撃、魔法など)                 |
| 予備行動 | 各キャラクターがターン中に実行できる1回の予備動作 (ポーションの使用、移動など) |

##### 行動

- 攻撃や魔法・奇跡の行使などは「行動」として実行する

行動に該当する行為:

| 行動         | 内容                                               |
| :----------- | :------------------------------------------------- |
| 攻撃         | 武器で射程内のエリアの対象を攻撃する               |
| 魔法・奇跡   | スペルを行使する                                   |
| アイテム使用 | 携行しているアイテムを使用する                     |
| 攻撃準備     | 次の攻撃を準備する                                 |
| 防御姿勢     | 敵の攻撃に備える                                   |
| 警戒指示     | 特定の敵への警戒を指示する                         |
| 防具の着脱   | 防具1つを着脱する                                  |
| その他       | GMと相談のうえ、上記に当てはまらない行動を実行する |

##### 予備行動

- エリアの移動、ポーション、スクロール、一部のアイテムやスペルは「予備行動」として、行動とは別に実行できる
- 予備行動は、各キャラクターにつき、1ターンに1回だけ実行できる
- 通常の行動と組み合わせて実行することができる
    - 例: ポーションの使用 → 攻撃
- 通常の行動として実行することもできる
    - 例: ポーションの使用 → ポーションの使用
- 通常の行動の後に実行することもできる
    - 例: 攻撃 → ポーションの使用

予備行動に該当する行為:

| 予備行動                       | 内容                                                                     |
| :----------------------------- | :----------------------------------------------------------------------- |
| 移動                           | 隣接したエリアに移動する                                                 |
| ポーション/スクロールの使用    | 携行しているポーション/スクロールを使用する                              |
| 「予備行動可」のスペルの行使   | 一部のスペルは、予備行動として行使できる                                 |
| 「予備行動可」のアイテムの使用 | 一部のアイテムは、予備行動として使用できる                               |
| 落ちているアイテムを拾い上げる | 自身のエリアに落ちているアイテムは、予備行動として拾い上げることができる |
| アイテムを受け渡す             | 自身と同じエリアにいる仲間に対してアイテムを受け渡す                     |
| アクセサリの着脱               | アクセサリ1つを着脱する                                                  |
| その他                         | GMと相談のうえ、上記に当てはまらない予備行動を実行する                   |

補足:
- 予備行動は、1ターンに1回までしか実行できない
    - ポーションの使用 → スクロールの使用 → 攻撃 はNG
- ポーションの投擲は、予備行動での実行は不可
    - 投擲は攻撃扱いとなるため、通常の行動として実行する
- 予備行動として武器や盾を召喚する魔法や奇跡を行使した場合、その行使の中で対象にアイテムを手渡したこととしても良い
    - 例: 予備行動で「フロストソード」により武器を召喚し、射程:0以内の味方に武器を手渡す → 予備行動だけで実行可能。自身の行動として別の行為を実行可能

##### 行動に当たらないが、自身のターンでしかできないこと

- 武器の着脱と持ち替え
    - 手に持っている武器を仕舞う
    - 携行している武器を手に持つ
    - この操作は、自身のターンで何回でも実行して良い
    - 武器以外の装備品の切り替えは、予備行動ではなく自身の行動で実行する必要がある
- 1H←→2Hの切り替え
    - 1H/2Hと記載のある武器は、自身のターン内で自由に切り替えることができる
    - この操作は、自身のターンで何回でも実行して良い
    - 自身のターン終了時の持ち方を、次の手番まで維持する
    - 例: 1Hで持っていたロングソードを2Hに切り替えて攻撃後、そのターンで1Hに戻しても良い
        - この場合、次の手番までロングソードは1Hとして装備した状態となる

---

#### 2-4. 射程

- 各行動や攻撃手段、魔法、アイテムには射程が存在する

| 射程       | 届く範囲          | 例                                   |
| :--------- | :---------------- | :----------------------------------- |
| 0 / 接触   | 同じエリア        | 味方前衛 → 味方前衛                  |
| 1 / 近距離 | 隣のエリアまで    | 味方前衛 → 敵前衛                    |
| 2 / 遠距離 | 2つ隣のエリアまで | 味方後衛 → 敵前衛、味方前衛 → 敵後衛 |
| 3 / 長距離 | 3つ隣のエリアまで | 味方後衛 → 敵後衛                    |

> 射程の解釈について:
> - 剣などの一般的な攻撃手段は、射程:1であるため隣のエリアに攻撃することができる
> - 1エリアの幅は10m、エリア間は密接しているため、近接攻撃をする際は、キャラクターと攻撃対象とはエリアの境目付近で接触している状態となる
> - 互いにエリアの後部ギリギリに立っていると20mの距離が開くことになるが、戦闘の進行上ではこうした細かな立ち位置については考慮しなくて良い
>     - ロールプレイとして、キャラクターの細かな立ち位置や距離を表現しても良い
>     - ゲームの進行上では、そのエリアにいるキャラクターは、常にそのエリアと射程のルールに従う

---

### 3. 戦闘終了

- 以下のいずれかの条件を満たしたとき、戦闘は終了する
    - いずれかの陣営が全滅した
    - いずれかの陣営が撤退を宣言し、成功した
    - GMが戦闘終了を宣言した (降伏、イベント等)

> 全滅の判定:
> - その陣営のキャラクターがすべて気絶、死亡、撤退、その他の理由で戦闘不能となっている状態
> - 高強度の魅了や凍結によって、実質的に戦闘行為ができない場合も、その陣営が全滅したものとして扱う

## HPとダメージのルール

- 戦闘ではキャラクターはHPの増減が生じる
- キャラクターは防具などで「装甲」を得たり、魔法や奇跡の効果で「障壁」を獲得できる
- ダメージを受けると $障壁 → 装甲 → HP$ の順に点数が減少していく
- HPが尽きると、キャラクターは気絶したり、死亡したりする
- ルールの詳細は [./01-hp-and-damage.md](./01-hp-and-damage.md) を参照すること

---

## 攻撃と防御のルール

- 戦闘中、キャラクターは自身が持つ攻撃手段を用いて、敵を攻撃する
- 攻撃したキャラクターは「攻撃判定」で自身の攻撃のダメージを決定する
- 攻撃を受けたキャラクターは「回避判定」で自身が受けるダメージを軽減する
- ルールの詳細は [02-attack-and-defence.md](./02-attack-and-defence.md) を参照すること

---

## 移動と包囲のルール

- 戦闘中、キャラクターはエリア間を「移動」できる
- 敵陣で数的不利が生じた場合、キャラクターは「包囲」を受ける
- ルールの詳細は [03-moving-and-placement.md](./03-moving-and-placement.md) を参照すること

---

## 属性と状態異常のルール

- 攻撃手段よっては「属性」が付与されていたり、相手に「状態異常」を与えたりすることができる
- エネミーや種族、装備により、特定の属性の効果が高まったり、その属性で受けるダメージが増加・減少したりする
- 状態異常を受けたキャラクターには、様々な悪効果が生じる
- ルールの詳細は [04-element-and-abnormal-condition.md](./04-element-and-abnormal-condition.md) を参照すること

---

## 戦闘中における魔法・奇跡のルール

- 戦闘中も [../04-spells/](../04-spells/) のルールに従い、各スペルを行使できる
- 効果に「予備行動可」と記載されているスペルは、予備行動として行使できる
- 魔法・奇跡によるダメージは抵抗不可であり、回避や防護による軽減ができない
    - 効果に「回避可能」や「防護可能」と記載されているスペルについては、攻撃と同様に軽減することができる
- 魔法・奇跡よにって付与された状態異常は、その強度に対して抵抗ロールを行うことができる

- ルールの詳細は [05-common-actions.md](./05-common-actions.md) を参照すること

## 戦闘後の処理

- 戦闘後、PCは下記の処理を任意に実行できる
    1. 戦利品の獲得: 戦利品判定 (10分所要)
    2. 武具の修理: 修理判定 (1時間所要)
    3. 傷の手当: 治療判定 (1時間所要)
- それぞれの判定については、 [../06-adventuring/01-exploration.md](../06-adventuring/01-exploration.md) を参照すること

> 戦闘後以外でも実行可能:
> - 修理判定や治療判定については、戦闘終了後のタイミングでなくても、任意に実行して良い
>     - どの状況でも、実行には1時間を所要する点に注意すること
> - 戦利品判定は、エネミーのドロップ品の獲得のために行うが、それ以外の戦利品に対して用いても良い
>     - 例えば宝物庫からめぼしいアイテムを探す場合や、宝箱の中身を占う場合にも、戦利品判定を用いて良い
> - これらの判定は、時間を短縮しての判定実施が可能である
>     - GMはセッションやシナリオの流れから、時間を短縮した判定をPLに実行させても良い
