# 判定のルール

- このドキュメントでは、プレイ中にダイスロールによる判定を行う際の手順について解説します

## 判定

- 探索中に発生する判定は、GMが状況に応じて適切な能力値と目標値を指定する
- PLは指定された能力値で NM6 をロールし、MtV が目標値以上なら成功

> 代用判定:
> - PLが「この状況ならこの能力値のほうが適切では？」と提案した場合、GMが妥当と判断すれば別の能力値での判定を認めて良い
> - 背景によるロールプレイボーナス (+2d) も同様に、PLの提案に基づきGMが判断する

---

## 判定一覧

### 生命力判定 (CON)

- 毒や病気に対する対抗に使用する

| 目標値 | 状況の目安                           |
| -----: | :----------------------------------- |
|      2 | 風邪、破傷風、腐った食材             |
|      3 | 疫病、毒性植物、生物毒、下水を浴びる |
|      4 | 強酸、致死性の毒キノコ               |
|      5 | 神話級の毒、極めて危険な毒           |

### 精神力判定 (MND)

- 魅了や精神操作に対する抵抗に使用する

| 目標値 | 状況の目安                                           |
| -----: | :--------------------------------------------------- |
|      2 | 眠気、アルラウネの魅了、愛妻からの懇願、空腹時の食事 |
|      3 | 睡魔、ラミアの魅了、空腹時のごちそう                 |
|      4 | 強力な暗示、精神操作                                 |
|      5 | 神話級の誘惑や精神干渉                               |

### 対人判定 (APP)

- NPCとの交渉、説得、値切り、情報の聞き出しなどに使用する

| 目標値 | 状況の目安                                         |
| -----: | :------------------------------------------------- |
|      2 | 友好的な相手への普通のお願い                       |
|      3 | 中立的な相手への交渉、ちょっとした無理のあるお願い |
|      4 | 警戒している相手の説得、かなり厚かましいお願い     |
|      5 | 敵対的な相手との交渉                               |

### 探索判定 (DEX / INT / LUC から選択)

- 周囲の観察、手がかりの発見、目星、聞き耳などに使用する
- PLは DEX、INT、LUC のいずれかを選んでロールできる。GMはシチュエーションに応じて使用可能な能力値を制限してもよい
    - DEX: 手で触れて調べる、細かい部分を指先で確認する
    - INT: 論理的に推理する、知識から手がかりを導く
    - LUC: 偶然の発見、なんとなく目についた

| 目標値 | 状況の目安                                         |
| -----: | :------------------------------------------------- |
|      2 | 注意して見れば気づく。隠されていない物品の発見     |
|      3 | 意識して探さないと見落とす。隠し扉、小さな手がかり |
|      4 | 巧妙に隠されている。精巧な隠し部屋、暗号の手がかり |
|      5 | 極めて困難。魔法で隠蔽された物、古代文明の秘密     |

### 解除判定 (DEX)

- 鍵開け、罠の解除、仕掛けの操作などに使用する

| 目標値 | 状況の目安                   |
| -----: | :--------------------------- |
|      2 | 単純な鍵、見えている罠の解除 |
|      3 | 一般的な錠前、標準的な罠     |
|      4 | 精巧な錠前、複雑な仕掛け     |
|      5 | 名工の錠前、魔法的な仕掛け   |

### 感知判定 (DEX / LUC から選択)

- 索敵、奇襲の察知、気配の感知などに使用する
- PLは DEX、LUC のいずれかを選んでロールできる
    - DEX: 反射的に気配を察知する、物音に反応する
    - LUC: ふと嫌な予感がする、なんとなく振り返った

| 目標値 | 状況の目安                                 |
| -----: | :----------------------------------------- |
|      2 | 隠れようとしていない敵の発見、明らかな異変 |
|      3 | 隠れている敵の発見、待ち伏せの察知         |
|      4 | 巧妙に潜んでいる敵、魔法による隠蔽         |
|      5 | 透明化した敵、ミミックの擬態 (高INT個体)   |

### 隠密判定 (DEX)

- 忍び足、尾行、身を隠すなどに使用する
- 感知判定と対抗が生じる場合がある

| 目標値 | 状況の目安                                         |
| -----: | :------------------------------------------------- |
|      2 | 暗がりでの隠れ、注意を払っていない相手への忍び寄り |
|      3 | 警戒している相手への忍び寄り、見張りの目を盗む     |
|      4 | 複数の見張りの目をかいくぐる                       |
|      5 | 極めて鋭い警戒の中での行動                         |

> 対抗判定:
> - 隠密側のMtVと感知側のMtVを比較する
> - 能動側が上回れば感知されない。同値なら感知側が勝利する

### 筋力判定 (STR)

- 重い物を動かす、扉を蹴破る、綱引き、拘束からの脱出などに使用する

| 目標値 | 状況の目安                                   |
| -----: | :------------------------------------------- |
|      2 | 重い扉を開ける、がれきを退かす               |
|      3 | 施錠された木の扉を蹴破る、人ひとりを引きずる |
|      4 | 鉄の扉をこじ開ける、岩を動かす               |
|      5 | 人間の限界に近い力仕事                       |

### 知識判定 (INT)

- アイテムの鑑定、碑文や文書の解読、魔法の識別、歴史的知識などに使用する

| 目標値 | 状況の目安                                         |
| -----: | :------------------------------------------------- |
|      2 | 一般常識で分かること。よくある品の鑑定             |
|      3 | 専門知識が要ること。珍しい品の鑑定、古い文書の解読 |
|      4 | 高度な学識。古代語の解読、魔法アイテムの識別       |
|      5 | 極めて稀な知識。失われた文明の記録、禁忌の術の識別 |

### 神学判定 (FTH)

- 神殿や聖遺物の鑑定、神聖語の解読、啓示の分析などに使用する

| 目標値 | 状況の目安                                             |
| -----: | :----------------------------------------------------- |
|      2 | 一般の信徒なら理解できること                           |
|      3 | 聖典の読解                                             |
|      4 | 秘匿されていることがらへの洞察                         |
|      5 | 極めて稀な知識。古の神の神殿や聖遺物の鑑定、奇跡の判別 |

### 運判定 (LUC)

- 上記のどれにも当てはまらない、運や偶然に左右される判定すべてに使用する
- ランダムに生じるイベント、敗北時の結末の決定など

| 目標値 | 状況の目安                             |
| -----: | :------------------------------------- |
|      2 | ちょっとした幸運。道端で薬草を見つける |
|      3 | そこそこの幸運。有用なアイテムの発見   |
|      4 | かなりの幸運。レアな発見               |
|      5 | 奇跡的な幸運                           |

---

### 汎用行動判定 (DEX)

- 登攀、跳躍、水泳、バランスなど、身体を使った行動全般はDEXで判定する

| 目標値 | 状況の目安                                           |
| -----: | :--------------------------------------------------- |
|      2 | 多少の身体能力があれば可能。梯子を登る、浅い川を渡る |
|      3 | 訓練が必要。崖を登る、急流を泳ぐ、細い足場を渡る     |
|      4 | 熟練者でも緊張する。断崖の登攀、激流の横断           |
|      5 | 人間の身体能力の限界。垂直の壁、嵐の海               |

---

### イニシアチブ判定 (LUC)

- 戦闘開始時、すべてのPCがLUCでロールする
- 目標値は敵のイニシアチブ値とし、MtV が目標値以上なら PC 側が先攻、未満なら敵側が先攻となる
- 詳細は [../07-combat/](../07-combat/) を参照すること

| 目標値 | 状況の目安                                                     |
| -----: | :------------------------------------------------------------- |
|      0 | このものはこちらを能動的に襲うことができない                   |
|      1 | 緩慢な相手。戦闘を試みたら、まず確実に先手を取れる             |
|      2 | 標準的な相手。準備をすれば先手を取りやすい                     |
|      3 | 素早かったり、狡猾な相手。正面からの戦いでは先手を取られやすい |
|      4 | かなり素早い相手。先手を取ることは非常に困難                   |
|      5 | 確実に先手を取られる相手。機先を制するには運も必要             |

> 偶発的遭遇:
> - 遭遇とはすべてが偶発的であり、その巡り合いの形は運によって定まる。
> - それがセッション上の計画的遭遇であったとしても、PCにとっては偶発的である。

### 攻撃判定 (DEX)

- 攻撃対象と攻撃手段を宣言し、攻撃判定で攻撃のダメージを決定し、攻撃を行う
- 攻撃のダメージは下記の計算式で定まる

$$
ダメージ = 武器の威力 + MtV
$$

- 詳細は [../07-combat/](../07-combat/) を参照すること

---

### 魔法行使判定 (INT)

- 魔法を行使する
- INT以下の魔法のみを行使可能
- 判定に使用するダイス数は、下記の計算式で定まる

$$
ダイス数 = INT - 難易度
$$

- ダイス数が0個以下となる場合、その魔法は行使できない
- ボーナスによりダイス数が1個以上となる場合でも、INTを超える難易度の魔法は行使できない
- 詳細は [../04-spells/](../04-spells/) を参照すること

---

### 奇跡行使判定 (FTH)

- 奇跡を行使する
- FTH以下の奇跡のみを行使可能
- 判定に使用するダイス数は、下記の計算式で定まる

$$
ダイス数 = FTH - 難易度
$$

- ダイス数が0個以下となる場合、その奇跡は行使できない
- ボーナスによりダイス数が1個以上となる場合でも、FTHを超える難易度の奇跡は行使できない
- 詳細は [../04-spells/](../04-spells/) を参照すること

### 修理判定 (DEX)

- 1時間かけて武具を修理する
- その装備の威力や防護が減少している場合、その効果を解除する
- 防具を修理した場合、減少した装甲をMtV-1点回復する

#### 短時間での修理判定

- 修理判定は1時間を所要するが、ペナルティを受けることで短い時間での修理獲得が可能

10分で修理判定を行う場合:
- ダイスに-2dのペナルティを受ける
- 装甲は1点までしか回復できない
- 判定後1時間、同じ対象に対して連続で修理判定を行うことはできない

---

### 治療判定 (DEX)

- 1時間かけて対象を治療する
- 対象のHPをMtV-1点回復する

#### 短時間での治療判定

- 治療判定は1時間を所要するが、ペナルティを受けることで短い時間での治療獲得が可能

10分で治療判定を行う場合:
- ダイスに-2dのペナルティを受ける
- HPは1点までしか回復できない
- 判定後1時間、同じ対象に対して連続で治療判定を行うことはできない

---

### 戦利品判定 (LUC)

- 10分を所要し、対象から戦利品を獲得する
- 戦闘で撃破した敵や、略奪品の山、ダンジョンでの発見物などを対象に実行する
- 戦利品獲得は、対象1つに対し、1人のみ実行可能
- 戦利品表を参照し、MtVが目標値以上の戦利品をすべて獲得する

| 目標値 | レアリティの目安                                                 |
| -----: | :--------------------------------------------------------------- |
|      1 | 必ず獲得できる                                                   |
|      2 | 多くの場合獲得できる                                             |
|      3 | 運が良いと見つかる                                               |
|      4 | 基本的には得られない                                             |
|      5 | まず得られない戦利品。運の良い者が、さらに運に恵まれる必要がある |

> 例:
> - ゴブリンリーダーを倒した。代表者が LUC 4 で 4M6 → MtV 2
> - ドロップ表: ショートソード(目標1) / 小銭(目標2) / レザーシールド(目標3)
> - → MtV 2 なので、ショートソードと小銭を獲得
>
> 注意:
> - 戦利品の獲得には10分を要する
>     - 捜索・解体・回収の時間
> - 冒険中は、この間に襲撃が生じる場合もある
>     - 判定中に襲撃が生じた場合、戦利品判定を行っている者は感知判定に-2dのペナルティを受けるものとして扱う

#### 短時間での戦利品判定

- 戦利品判定は10分を所要するが、ペナルティを受けることで短い時間での戦利品獲得が可能

1分で戦利品判定を行う場合:
- ダイスに-2dのペナルティを受ける
- 目標値3以内の戦利品しか獲得できない

10秒で戦利品判定を行う場合:
- ダイスに-4dのペナルティを受ける
- 目標値2以内の戦利品しか獲得できない

> 扱いについて:
> - GMは基本的には通常の戦利品獲得をさせるが、PLが希望した場合は短時間での回収を認めても良い
> - 撤退戦や防衛戦など、戦利品獲得に使える時間がないシチュエーションなどでこのルールを適用すると良い

## 達成値が目標値を上回った場合

- 達成値が目標値を上回った場合は、その判定は成功となる
- 大きく上回っている場合、その判定はより大きな成功を掴んだものとして扱っても良い
- 逆に、目標値にわずかに届いていない場合も、GMの裁量によって、小さな情報を与えても良い

> 部屋の探索のために目標値3の探索判定を行う例:
> - MtV 1: 失敗。その判定では何も得られないか、誰にでも一目でわかる情報だけが得られる
> - MtV 2: 失敗。一目でわかる情報が得られる。さらに「何かが隠されていること」だけはわかった
> - MtV 3: 成功。この部屋の壁にかかった絵画の裏に、スイッチが隠されていた
> - MtV 4: 大成功。このスイッチは、どうやら隠し扉を開くスイッチであることがわかる
>     - 本来であれば、この後にスイッチに対する知識判定や探索判定を追加で行うところを、スキップできる
> 
> 罠の検出のために目標値3の感知判定を行う例:
> - MtV 1: 失敗。知らずのうちに罠が作動し、矢が飛来した。回避できない
> - MtV 2: 失敗。ワイヤートラップに引っかかり、矢が飛来した。回避を試行できる
> - MtV 3: 成功。足元に仕掛けられたワイヤートラップを感知した
> - MtV 4: 大成功。このワイヤートラップは、作動すると矢を放つ仕掛けのようだ
>     - トラップに対する知識判定をスキップできる
> - MtV 5: 大成功。このワイヤートラップの解除方法がわかる
>     - 解除判定をする際の目標値を-1する

## 協力判定

協力判定の流れ:
1. 協力判定を行う場合、主目的の判定を行う代表者を一人決定する
2. 協力者が、どう協力するのかを宣言する
3. GMはそれに応じ、協力者が用いる判定を決める
4. 協力者は指定の判定でロールを行い、MtVを求める
5. すべてのMtVを合計し、代表者は、その分だけボーナスダイスを得る
6. 代表者は自身の能力値を参照し、ボーナスダイスを加えて判定を行う

代表者のダイス数は下記の式で決定する:
$$
代表者のダイス数 = 代表者の能力値 + 代表者の修正 + 協力者の判定のMtVの合計
$$

協力判定のシチュエーションの例:
- 誰かが図書館で調査している時に、その調査をサポートする
- 重たい扉を、協力して押し開ける
- 重量物を、複数人で担いで支える

協力判定の条件:
- 誰か一人の判定をサポートする形でのみ、協力判定の実行が可能
    - 小さな複数の成功を積み重ねる判定ではなく、大きなひとつの成功が必要な判定で有効
- 分散・分担して行動している場合の判定は、協力判定ができない
    - 全員で分散して薬草を採取する
    - 部屋や通路を探索する
- 個人の素質に依存した判定は、協力判定ができない
    - 感知判定、運判定
    - 魔法行使判定、奇跡行使判定

> 図書館で目標値4の知識判定を行う例:
> - 4人PT。INTが最も高いPCが代表者となる
> - 協力者が3人は、それぞれ「同じ知識判定でサポートする」「文献を持ってくる速度、DEXで判定したい」「LUCが高いから、運良く目当ての本を探せないか」と提案する
> - GMはそれぞれに、知識判定、探索判定、運判定を許可する
> - それぞれのPCの判定の結果、MTVが2、2、3となる
> - 代表者は+7dのボーナスダイスを得て、知識判定を行うことができる
